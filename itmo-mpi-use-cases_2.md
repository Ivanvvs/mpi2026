Прецедент: CreateAndTakeExam
```
ID:1
```
Краткое описание:
Экзаменатор-модератор создает экзамен и управляет его статусом, а студенты проходят
экзамен и отправляют ответы. Система сохраняет ответы, контролирует статус экзамена
и после завершения сохраняет результаты экзамена.
Главные актеры:
Студент класса
Экзаменатор-модератор
Второстепенные актеры:
Нет.
Предусловия:
1. Экзаменатор-модератор авторизован в системе.
2. Студент класса авторизован в системе.
Основной поток:
1. Прецедент начинается, когда Экзаменатор-модератор выбирает опцию создания
экзамена.
2. Система запрашивает параметры экзамена.
3. Экзаменатор-модератор вводит параметры экзамена.
4. Система сохраняет экзамен и присваивает ему статус «Подготовлен».
5. Экзаменатор-модератор запускает экзамен.
6. Система изменяет статус экзамена на «Активен».
7. Система предоставляет Студентам доступ к экзамену.
8. Студент начинает прохождение экзамена.
9. Пока экзамен не завершен.
9.1. Студент добавляет ответы на экзаменационные вопросы.
9.2. Система сохраняет ответы Студента.
9.3. Экзаменатор-модератор отслеживает процесс проведения экзамена.
точка расширения: ViolationDetected
10. Если Экзаменатор-модератор завершает экзамен.
10.1. Система изменяет статус экзамена на «Завершен».
11. Если время экзамена истекает.
11.1. Система автоматически изменяет статус экзамена на «Завершен».
12. Система блокирует изменение ответов Студентов.
13. Экзаменатор-модератор выставляет итоговые баллы.
14. Система сохраняет результаты экзамена.
15. Система пересчитывает баланс S-очков классов.
16. Система обновляет сводную панель.
Постусловия:
1. Экзамен завершен.
2. Ответы Студентов сохранены в системе.
3. Итоговые баллы сохранены.
4. Статус экзамена изменен на «Завершен».
5. Баланс S-очков классов пересчитан.
6. Сводная панель обновлена.

Краткое описание:
Куратор класса инициирует тайное голосование среди Студентов класса. Студенты
анонимно отправляют голоса, а система сохраняет результаты голосования и
автоматически подсчитывает итоги.

Студент класса
Куратор класса

Нет.

2. Студент класса авторизован в системе.
Основной поток:
1. Прецедент начинается, когда Куратор класса выбирает опцию создания тайного
голосования.
2. Система запрашивает параметры голосования.
3. Куратор класса вводит параметры голосования.
4. Система создает голосование и присваивает ему статус «Активно».
5. Система предоставляет Студентам класса доступ к голосованию.
6. Студент класса открывает голосование.
7. Пока голосование не завершено.
7.1. Студент класса выбирает вариант голосования.
7.2. Система сохраняет голос Студента анонимно.
7.3. Система проверяет, что Студент класса еще не участвовал в голосовании.
8. Если Куратор класса завершает голосование.
8.1. Система изменяет статус голосования на «Завершено».
9. Если время голосования истекает.
9.1. Система автоматически изменяет статус голосования на «Завершено».
10. Система блокирует возможность повторного голосования.
11. Система автоматически подсчитывает результаты голосования.
12. Система сохраняет результаты голосования.
13. Система отображает итоги голосования Куратору класса.

1. Голосование завершено.
2. Голоса Студентов сохранены в системе.
3. Результаты голосования подсчитаны.
4. Статус голосования изменен на «Завершено».
